# 使用数据库

> **场景**：应用需要连接 PostgreSQL 或 Redis
> **预计时间**：20 分钟
> **前置条件**：已完成[部署第一个应用](./02.first-app.md)

---

## 概述

当前平台提供：

| 数据库 | 用途 | 访问方式 |
|--------|------|---------|
| **PostgreSQL** | 关系数据库 | Vault 静态密码 + Dokploy Env |
| **Redis** | 缓存/队列 | Vault 静态密码 + Dokploy Env |

> ClickHouse / ArangoDB 尚未落地，后续再补。

---

## 连接信息

### PostgreSQL

| 项目 | 值 |
|------|-----|
| **Host** | `platform-postgres` |
| **Port** | `5432` |
| **Vault 路径** | `secret/platform/prod/postgres` |

### Redis

| 项目 | 值 |
|------|-----|
| **Host** | `platform-redis` |
| **Port** | `6379` |
| **Vault 路径** | `secret/platform/prod/redis` |

> 前提：你的应用需要加入 `dokploy-network` 才能访问上述服务。

## 接入步骤

### 1. 获取凭据

由运维写入 Vault：
```bash
vault kv get -field=root_password secret/platform/prod/postgres
vault kv get -field=password secret/platform/prod/redis
```

### 2. 配置 Dokploy 环境变量

```bash
PG_HOST=platform-postgres
PG_PORT=5432
PG_USER=postgres
PG_PASS=<from Vault>

REDIS_HOST=platform-redis
REDIS_PORT=6379
REDIS_PASSWORD=<from Vault>
```

### 3. 在应用中使用

**Node.js (pg)**：
```javascript
import { Pool } from 'pg';

const pool = new Pool({
  host: process.env.PG_HOST,
  port: Number(process.env.PG_PORT || 5432),
  user: process.env.PG_USER,
  password: process.env.PG_PASS,
});
```

**Node.js (redis)**：
```javascript
import { createClient } from 'redis';

const client = createClient({
  url: `redis://:${process.env.REDIS_PASSWORD}@${process.env.REDIS_HOST}:${process.env.REDIS_PORT}`,
});
```

## 常见问题

### 无法访问数据库？
- 确认你的应用在 `dokploy-network` 上。
- 确认 Vault 中的密码已更新到 Dokploy Env。
