## 现状

平台的观测体系（SigNoz / OTel Collector）已落地并承载 finance_report 的生产流量（[Infra-007](../project/Infra-007.signoz_install.md)、[Infra-014](../project/Infra-014.finance_report_observability.md)）；OpenPanel 用户行为分析也已部署（[Infra-021](../project/Infra-021.openpanel_install.md)）。SigNoz 部署与运维细节见 [SigNoz Deployment Guide](./signoz.md)；采集架构与字段约束的权威定义见 [`docs/ssot/ops.observability.md`](../ssot/ops.observability.md)。

### OTel SDK 接入（后端，Docker 网络内）

前置：SigNoz 已部署健康；应用在 `dokploy-network` 内。

| 变量 | 说明 | 示例 |
|------|------|------|
| `OTEL_EXPORTER_OTLP_ENDPOINT` | OTLP HTTP 端点（所有环境同一地址） | `http://platform-signoz-otel-collector:4318` |
| `OTEL_SERVICE_NAME` | 服务名 | `finance-report-backend` |

Collector 的 4317/4318 仅 `expose` 于 Docker 网络，从不对外 `publish`。

### 浏览器前端 OTLP ingest（公网）

前端走唯一公网 ingest `otel.${INTERNAL_DOMAIN}`；运行时（非 build-time）注入：
`NEXT_PUBLIC_OTEL_EXPORTER_OTLP_ENDPOINT=https://otel.${INTERNAL_DOMAIN}/v1/traces`。

### UI 访问地址

- SigNoz Web UI：`https://signoz${ENV_DOMAIN_SUFFIX}.${INTERNAL_DOMAIN}`

### 告警规则

告警路径固定为 `component/app → OTLP Collector → SigNoz → platform/12.alerting → Feishu/Lark`；规则与阈值见 [`platform/12.alerting`](../../platform/12.alerting/README.md)。
