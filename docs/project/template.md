# Project Documentation Templates

Use these templates when creating or updating `docs/project/Infra-XXX.*.md` files.

## Project Template

```
# Infra-XXX: Project Title

**Status**: In Progress  
**Owner**: Infra  
**Priority**: P?  
**Branch**: `feat/your-branch` (optional)

## Goal
One sentence defining the completed state.

## Context
Why this project exists and the problem it solves.

## Scope
- [ ] Task 1
- [ ] Task 2

## Deliverables
- Deliverable A
- Deliverable B

## PR Links
- None yet.

## Change Log
| Date | Change |
|------|--------|
| YYYY-MM-DD | Initialized project |

## Verification
- [ ] `command` or validation checklist

## References
- [SSOT: topic.key](../ssot/template.md)
```

## TODOWRITE Template

```
# Infra-XXX: TODOWRITE (Short Name)

**Status**: Active  
**Owner**: Infra

## Purpose
Track top issues discovered during the project.

## Top Issues (Top 30)
- [ ] `path/to/doc.md`: issue summary
```

> **Note**: When archiving a project, merge the TODOWRITE content into the archived project file.
